# Release Notes

## v0.32.0 - 2026-07-01

### 改进

- 成本对账历史回补按 6 小时窗口拆分 step，降低单次采集跨度和失败重试成本。
- 成本对账在供应商不支持充值订单或兑换记录读取时降级为诊断信息，继续采集余额、用量和其他可用数据。
- 用量消耗从供应商会话导入时支持超过 1000 条自动分批入库。
- 调度失败原因增加结构化 metadata，明确自动登录是否已尝试、失败阶段和建议处理动作。
- 成本对账页面新增历史回补运行记录、step 明细、失败原因弹窗、请求/结果快照查看，以及 step 重试/取消操作。
- 供应商列表刷新余额后只更新当前行，减少整表重载。

### 修复

- 修正会话缺失类调度失败被过早归类为人工处理的问题，允许自动登录或后台重试继续处理。
- 修正成本对账单供应商同步仍走旧 provision job 轮询的问题，统一进入调度器回补运行视图。

### 测试

- 增加可选成本能力缺失、用量分批导入、成本回补窗口拆分、自动登录失败诊断和后台重试路径回归测试。

### 发布

- 更新版本号到 `0.32.0`。
- GitHub Release 继续只发布 Linux 产物：`linux_amd64`、`linux_arm64` 和 `checksums.txt`。
