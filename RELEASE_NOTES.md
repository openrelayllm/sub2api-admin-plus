# Release Notes

## v0.34.0 - 2026-07-03

### 改进

- 供应商账号、供应商绑定、本地账号、账号运行态和用量汇总改为自动拉取所有分页，避免页面只显示前 1000 条数据。
- 调度器 Step 列表接口会附带 attempt 日志，成本回补页面和调度详情弹窗可在 step reason 为空时展示真实失败原因。
- 成本回补失败详情新增 attempt 日志区域，展示 attempt 序号、状态、耗时、错误码、错误信息和响应快照。
- 调度中心失败诊断统一使用 reason、attempt 日志和 result snapshot 组合生成摘要，提高排障可读性。
- 供应商账号分组展示改为优先显示本地账号分组，并单独标注供应商分组，减少倍率和供应商组信息混杂。

### 新增

- 新增通用 `loadAllPagedItems` 前端分页工具，用于复用全量分页加载逻辑。
- 新增前端分页工具和调度诊断展示回归测试。

### 修复

- 修正供应商账号和调度相关页面在数据超过单页上限时遗漏后续分页的问题。
- 修正成本回补 Step 只有 attempt 失败日志、没有 step reason 时，页面显示为无诊断的问题。
- 调整 Mihomo runtime 测试等待逻辑，避免日志写入异步导致本地测试偶发失败。

### 测试

- 增加调度 Step attempt 日志附加、前端失败诊断摘要、分页工具和 Mihomo runtime 日志等待的回归测试。

### 发布

- 更新版本号到 `0.34.0`。
- GitHub Release 继续只发布 Linux 产物：`linux_amd64`、`linux_arm64` 和 `checksums.txt`。
- DockerHub/Railway 镜像渠道不随常规发布自动执行；如需要，单独走镜像渠道发布流程。
